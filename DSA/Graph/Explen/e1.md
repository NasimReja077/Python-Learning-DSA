## Graph Adjacency List — Line by Line Explanation

---

### Class Definition & Constructor

```python
class Graph:
```
Creates a class called `Graph` to represent an **undirected graph**.

---

```python
def __init__(self):
    self.adjList = {}
```
Constructor initializes an **empty dictionary** called `adjList`.
Each **key** = a vertex, each **value** = list of its neighbors.

```
adjList = {}   ← starts completely empty
```

---

### add_vertex Method

```python
def add_vertex(self, vertex):
    if vertex not in self.adjList:
        self.adjList[vertex] = []
```
Adds a new vertex to the dictionary **only if it doesn't already exist**, and assigns it an empty list `[]` to hold future neighbors. Prevents overwriting existing connections.

---

### addEdge Method

```python
def addEdge(self, src, dest):
    self.add_vertex(src)
    self.add_vertex(dest)
```
Before adding an edge, **ensures both vertices exist** in the dictionary by calling `add_vertex` for each. If they already exist, nothing changes.

---

```python
self.adjList[src].append(dest)
self.adjList[dest].append(src)
```
Connects the two nodes **in both directions** (undirected graph):
- `src`'s list gets `dest` added
- `dest`'s list gets `src` added

---

### printGraph Method

```python
def printGraph(self):
    for vertex in self.adjList:
        print(vertex, " -> ", self.adjList[vertex], end="\n")
```
Loops through every key in the dictionary and prints each vertex with its **list of neighbors**.

## Step-by-Step — How adjList Builds Up

```
g.addEdge(1, 2) → { 1:[2],        2:[1] }
g.addEdge(2, 3) → { 1:[2],        2:[1,3],    3:[2] }
g.addEdge(1, 4) → { 1:[2,4],      2:[1,3],    3:[2],    4:[1] }
g.addEdge(4, 3) → { 1:[2,4],      2:[1,3],    3:[2,4],  4:[1,3] }
g.addEdge(2, 4) → { 1:[2,4],      2:[1,3,4],  3:[2,4],  4:[1,3,2] }
g.addEdge(4, 5) → { 1:[2,4],      2:[1,3,4],  3:[2,4],  4:[1,3,2,5], 5:[4] }
g.addEdge(3, 5) → { 1:[2,4],      2:[1,3,4],  3:[2,4,5],4:[1,3,2,5], 5:[4,3] }
```

## Adjacency Matrix vs Adjacency List

| Feature | Adjacency Matrix | Adjacency List |
|---|---|---|
| Storage | O(V²) — fixed grid | O(V+E) — only stores edges |
| Best for | Dense graphs | Sparse graphs |
| Edge lookup | O(1) | O(degree) |
| Structure | 2D list | Dictionary of lists |

> **Key Rule:** Since the graph is **undirected**, every edge `(A, B)` is stored **twice** — once in A's list and once in B's list, keeping the structure always **symmetric**.
