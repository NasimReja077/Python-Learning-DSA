🔷 Algorithm: Kruskal’s Algorithm
Algorithm: KRUSKAL(G)

Input: 
    G = (V, E)  → Weighted Undirected Graph

Output:
    Minimum Spanning Tree (MST)

Step 1: Start

Step 2: Create an empty set MST

Step 3: Sort all edges of graph in increasing order of weight

Step 4: Make a separate set for each vertex 
        (Using Disjoint Set / Union-Find)

Step 5: For each edge (u, v) in sorted order:

            a. Find the root of u and v
            b. If root(u) ≠ root(v)
                    Add edge (u, v) to MST
                    Union(u, v)
            c. Else
                    Discard the edge (to avoid cycle)

Step 6: Repeat until MST contains (V − 1) edges

Step 7: Print MST

Step 8: Stop
-----
Cycle Detection (Union-Find Algorithm)

FIND(parent, i):
    If parent[i] == i
        Return i
    Else
        Return FIND(parent, parent[i])

UNION(parent, x, y):
    parent[FIND(x)] = FIND(y)
-----

📌 Important Points ⭐
Greedy Algorithm
Used for Weighted Undirected Graph
Sort edges first
Avoid cycles using Union-Find
MST contains exactly (V − 1) edges
Time Complexity = O(E log E)
Space Complexity = O(V)

-------

Kruskal’s Algorithm sorts all edges in increasing order and selects the smallest edge that does not form a cycle until (V−1) edges are selected. Cycle detection is done using Disjoint Set (Union-Find). Time complexity is O(E log E).

🔷 Algorithm: Prim’s Algorithm (Using Adjacency Matrix)

Algorithm: PRIM(G)

Input:
    G = Weighted Undirected Graph
    V = Number of vertices

Output:
    Minimum Spanning Tree (MST)

Step 1: Start

Step 2: Create arrays:
        key[V] = ∞ (store minimum weight)
        parent[V] (store MST)
        visited[V] = False

Step 3: Choose starting vertex (usually 0)
        key[0] = 0
        parent[0] = -1

Step 4: Repeat V times:
        a. Select vertex u with minimum key value 
           which is not yet visited
        b. Mark u as visited
        c. For each adjacent vertex v of u:
                If not visited[v] AND
                   weight(u, v) < key[v]
                Then
                   key[v] = weight(u, v)
                   parent[v] = u

Step 5: Print edges using parent array

Step 6: Stop

-------

📌 Important Points ⭐

Greedy Algorithm
Works on Weighted Undirected Graph
Builds single tree
Selects smallest edge from visited → unvisited
MST contains (V − 1) edges
No cycle formed
Time Complexity:
Using Matrix → O(V²)
Using Min Heap → O(E log V)
Space Complexity → O(V)

---

Prim’s Algorithm is a greedy method that starts from any vertex and repeatedly selects the minimum weight edge connecting a visited vertex to an unvisited vertex until (V−1) edges are selected. It produces a Minimum Spanning Tree. Time complexity is O(V²).

🌳 Algorithm: Path Matrix using Warshall’s Algorithm
🔷 Input:

Adjacency Matrix A[n][n]

🔷 Output:

Path Matrix P[n][n] (Transitive Closure)

✅ Full Algorithm
Algorithm: WARSHALL(A, n)

Step 1: Start

Step 2: Initialize P = A
        (Copy adjacency matrix into P)

Step 3: For k = 0 to n-1 do
            For i = 0 to n-1 do
                For j = 0 to n-1 do

                    If P[i][j] == 1
                        Continue

                    Else if P[i][k] == 1 AND P[k][j] == 1
                        Set P[i][j] = 1

Step 4: Return P

Step 5: Stop


🔷 Short Version (Write in Exam Quickly)
For k = 0 to n-1
   For i = 0 to n-1
      For j = 0 to n-1
         P[i][j] = P[i][j] OR (P[i][k] AND P[k][j])

🔷 Time Complexity
✔ O(n³)

🔷 Important Points ⭐

Used to find Transitive Closure
Works on Directed Graph
Based on Dynamic Programming
Uses logical AND & OR operation


Warshall’s Algorithm computes the Path Matrix by checking indirect connections between all pairs of vertices using intermediate vertices. It runs in O(n³) time.

✅ Algorithm: Create Graph Using Adjacency List
Algorithm: CREATE_GRAPH(V, E)

Step 1: Start

Step 2: Create an array/list Adj of size V
        (Each element is an empty list)

Step 3: For each edge (u, v) in E do:

            a. Add v to Adj[u]

            b. If graph is Undirected:
                    Add u to Adj[v]

Step 4: Stop


🔷 Time & Space Complexity

Space Complexity → O(V + E)
Adding Edge → O(1)
Best for Sparse Graphs

⭐ Important Points

✔ Uses list/array of lists
✔ Saves memory compared to matrix
✔ Good for large sparse graphs
✔ Faster traversal (O(V + E))


Adjacency List represents a graph using an array of lists where each list stores adjacent vertices. It requires O(V + E) space and is efficient for sparse graphs.

🌳 Algorithm: Adjacency List Representation (Using Dictionary)
🔷 Input:

Edges of the graph

🔷 Output:

Graph stored in adjacency list format

✅ Algorithm: CREATE_GRAPH()
Step 1: Start

Step 2: Create an empty dictionary AdjList

Step 3: For each edge (src, dest) do:

            a. If src is not in AdjList:
                    Add src with empty list

            b. If dest is not in AdjList:
                    Add dest with empty list

            c. Add dest to AdjList[src]

            d. Add src to AdjList[dest]   (For Undirected Graph)

Step 4: To print graph:
            For each vertex in AdjList:
                Print vertex and its adjacency list

Step 5: Stop


🔷 Time Complexity

Adding Vertex → O(1)
Adding Edge → O(1)
Space Complexity → O(V + E)

⭐ Important Points

✔ Uses Dictionary (Hash Map)
✔ Efficient for Sparse Graph
✔ Easy to insert edges
✔ Requires O(V + E) space


Adjacency List stores graph using a dictionary where each vertex contains a list of its adjacent vertices. For undirected graphs, both vertices are added to each other's list.

🌳 Adjacency Matrix Representation of Graph
🔷 Definition

Adjacency Matrix is a method of representing a graph using a 2D array (matrix) of size V × V, where:

If there is an edge between vertex i and j,
→ Matrix[i][j] = 1

Otherwise,
→ Matrix[i][j] = 0

🔷 Algorithm: CREATE_ADJ_MATRIX(V, E)
Step 1: Start

Step 2: Create a 2D array Matrix of size V × V

Step 3: Initialize all elements of Matrix to 0

Step 4: For each edge (u, v) in E do:

            a. Set Matrix[u][v] = 1

            b. If the graph is Undirected then
                   Set Matrix[v][u] = 1

Step 5: Stop


🔷 Explanation of Terms

V → Number of Vertices
E → Number of Edges
(u, v) → Edge connecting vertex u and v
Matrix[u][v] = 1 → Edge exists


🔷 Time & Space Complexity

Space Complexity → O(V²)
Edge Insertion → O(1)
Edge Checking → O(1)
