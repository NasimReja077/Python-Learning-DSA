What is a Minimum Spanning Tree?

Real World Analogy

Imagine you need to wire internet cables between 5 office buildings. Each possible cable route between two buildings has a cost (digging, cable length, etc). You want all buildings connected while spending the minimum total money. You also can't create loops (wasted cable). That's exactly what an MST gives you — the cheapest connected network with no redundancy.

🌐

Spanning Tree

A subgraph that connects ALL V vertices using exactly V−1 edges with no cycles. "Spanning" = covers every vertex.

⚖️

Minimum

Among all possible spanning trees, we want the one with the smallest total edge weight. Multiple valid MSTs can exist if weights tie.

🚫

No Cycles

Adding any edge to a spanning tree creates a cycle — making it no longer a tree. We use exactly V−1 = 4 edges for 5 vertices.

🔗

Connected

Every vertex must be reachable from every other vertex. Prim's works only on connected graphs (no isolated vertices).

Core Idea — How Prim's Works

The Algorithm in Plain English

Start at any vertex (vertex 0). Maintain two groups: vertices IN the MST and vertices NOT YET IN it.

At every step, look at all edges that cross the boundary (one end IN, one end OUT). Pick the cheapest crossing edge. Add that edge and its new vertex to the MST.

Repeat until all vertices are IN. That's your MST.

1
Initialize: Set key[0]=0, all others=∞, no vertex visited. key[] represents "how cheaply can I pull this vertex into the MST?"
2
Pick minimum key: Among unvisited vertices, pick the one with smallest key[]. That's the vertex we add to MST this round.
3
Mark visited: The chosen vertex u is now part of the MST. We'll never reconsider it (greedy lock-in).
4
Relax neighbors: For every neighbor v of u, if the edge u→v is cheaper than the current best cost to reach v (key[v]), update key[v] and parent[v].
5
Repeat: Go back to step 2 until all V vertices are visited. The parent[] array now encodes the MST edges.
03

The Three Core Arrays — What Each One Does

key[ ]
int[V] — initialized to sys.maxsize (∞)
Stores the minimum edge weight needed to connect each vertex to the growing MST. Think of it as "what's the cheapest ticket to get vertex v into the MST right now?"
key[0] = 0 # free, we start here
key[1] = ∞ # not reachable yet
key[1] = 2 # after finding edge 0→1
parent[ ]
int[V] — initialized to None
Records which vertex in the MST connects to each vertex. When we finalize the MST, we read this array to know which edges were chosen. This IS the MST.
parent[0] = -1 # root, no parent
parent[1] = 0 # vertex 1 connects via 0
parent[2] = 1 # vertex 2 connects via 1
visited[ ]
bool[V] — initialized to False
Tracks which vertices are already in the MST. Once a vertex is marked visited, its key[] and parent[] values are finalized and never change.
visited[0] = True # added to MST
visited[1] = False # still outside MST
visited[1] = True # after picking vertex 1

Adjacency Matrix — The Data Structure

mat[i][j] = weight of edge between i and j. mat[i][j]=0 means no edge.
Matrix is symmetric since graph is undirected. MST edges are highlighted green.

prim_mst.py — min_key() — The Bottleneck 
 1def min_key(self, key, visited):
 2  # This linear scan is why the algorithm is O(V²) overall
 3  # A min-heap would reduce this to O(log V) per call
 4  min_val   = sys.maxsize
 5  min_index = -1
 6  for v in range(self.size):     # scan ALL V vertices
 7    if key[v] < min_val and not visited[v]:O(V) per call × V calls = O(V²)
 8      min_val   = key[v]
 9      min_index = v
10  return min_index               # index of cheapest unvisited vertex

Time & Space Complexity

PARTRUNSCOST EACHTOTAL
Outer loop V times — V
min_key() scan V times (once per outer loop) O(V) O(V²)
Inner neighbor loop V times (once per outer loop) O(V) O(V²)
Total Time O(V²) + O(V²) O(V²)
Space Matrix (V×V) + 3 arrays (V each) O(V²)
How to make it faster → O(E log V)
Replace the linear min_key() scan with a min-heap (priority queue).
Replace the adjacency matrix with an adjacency list.

→ Extracting minimum: O(log V) instead of O(V)
→ Scanning neighbors: O(degree) instead of O(V)
→ Total: O(E log V) — much faster for sparse graphs where E << V²
